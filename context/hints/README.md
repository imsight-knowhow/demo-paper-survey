# hints/ 目录说明（技巧与排错指南）

## HEADER
- **Purpose**: 汇总开发过程中的经验、常见问题与排错步骤，帮助快速解决重复问题
- **Status**: Active
- **Date**: 2025-11-14
- **Dependencies**: 上层 context/ 目录约定
- **Target**: 所有开发者与 AI 助手

## 内容说明

`hints/` 目录适合存放：

- 带有 `howto-` 前缀的操作指南（如环境搭建、调试方法）
- 常见错误原因说明（`why-this-error-occurs`）
- 故障排查手册（`troubleshoot-*`）

这些文档帮助后续的人类或 AI 避免「踩同样的坑」，提升开发与调试效率。
