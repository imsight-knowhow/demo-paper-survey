# design/ 目录说明（设计文档）

## HEADER
- **Purpose**: 存放系统架构、模块设计、接口规范等技术设计类文档
- **Status**: Active
- **Date**: 2025-11-14
- **Dependencies**: 上层 context/ 目录约定
- **Target**: 架构设计者、后端/前端开发者、AI 助手

## 内容说明

`design/` 用于记录与「如何设计系统」相关的内容，例如：

- 整体架构与模块划分
- 数据流、调用链与时序图
- 关键模块（如 agents、pipeline、存储层等）的技术方案
- API 规格说明与约束

建议每份设计文档都以 HEADER 开头，并在内容中给出足够的上下文，方便 AI 助手快速理解和引用。
