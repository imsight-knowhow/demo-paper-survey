# tasks/backlog/ 目录说明（待办任务）

## HEADER
- **Purpose**: 记录尚未开始或尚未排期的任务，作为未来迭代的候选项
- **Status**: Active
- **Date**: 2025-11-14
- **Dependencies**: plans/ 规划、产品需求
- **Target**: 产品/项目负责人、规划阶段的 AI 助手

## 内容说明

`backlog/` 主要用于：

- 收集未来可能实现的功能或改进想法
- 在正式推进前记录初步思路与风险
- 为版本规划提供候选任务集合

在任务进入执行阶段时，可将对应文件移动到 `tasks/working/`。
