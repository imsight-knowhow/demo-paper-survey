# tasks/ 目录说明（任务管理）

## HEADER
- **Purpose**: 管理项目任务的状态与描述，形成简单的文本看板
- **Status**: Active
- **Date**: 2025-11-14
- **Dependencies**: plans/ 规划文档
- **Target**: 项目维护者、执行任务的开发者与 AI 助手

## 内容说明

`tasks/` 目录下按状态划分子目录：

- `backlog/`：尚未开始但已记录的任务
- `working/`：当前正在执行的任务
- `done/`：已经完成并归档的任务

单个任务通常以 Markdown 文件形式存在，可包含：

- 任务背景与目标
- 具体子任务与验收标准
- 执行记录与更新
