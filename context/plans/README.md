# plans/ 目录说明（规划与实现路线）

## HEADER
- **Purpose**: 存放功能规划、实现路线图和拆解计划，指导后续迭代
- **Status**: Active
- **Date**: 2025-11-14
- **Dependencies**: design/ 文档、tasks/ 任务列表
- **Target**: 产品规划者、开发者、AI 助手

## 内容说明

`plans/` 中可以包括：

- 功能或版本级别的路线图（roadmap）
- 大型重构或新模块的实施计划
- 各阶段里程碑与验收标准

文件命名可以结合目标，如：

- `api-redesign-roadmap.md`
- `database-migration-strategy.md`
- `v2-implementation-plan.md`
